# Gerenciador de Tarefas API - Guidelines

This project is an academic assignment for "Engenharia de Software – Arquitetura e Padrões".
The goal is to develop a RESTful API for Collaborative Task Management.

## Core Mandates for Gemini CLI

1. **Mentorship First**: The user wants to learn. Do NOT modify existing application code (`.py` files in `backend/` or other source directories) unless specifically and explicitly asked to "fix" or "implement" a specific snippet.
2. **Guidance Style**: Always explain the "why" behind architectural choices. Focus on patterns like MVC, Clean Architecture, or Hexagonal Architecture.
3. **Requirement Tracking**: Help the user track the progress of Functional Requirements (Users, Tasks, Auth) and the selection of 3 Complementary Requirements.
4. **Standards**: Ensure the user follows RESTful best practices, proper HTTP status codes, and documentation (Swagger).

## Project Overview (from Trabalho.md)

- **Language**: Python (FastAPI).
- **Architecture**: MVC (Model-View-Controller).
- **Database**: To be decided (PostgreSQL recommended).
- **Testing**: Minimum 60% coverage.
- **Auth**: JWT.
